## 海克斯灵魂选择器 MVP 版本

**版本**：1.0  
**日期**：2026年4月1日  
**状态**：草案  
**负责人**：王达迅

## 1. 产品概述

### 1.1 产品愿景
通过游戏化的选择题，帮助用户发现自己的文艺灵魂，获得个性化深度推荐。

### 1.2 MVP范围
**版本1.0只包含**：
1. 四步海克斯选择器（完整流程）
2. 基础灵魂画像生成
3. 简单推荐展示
4. 用户选择记录

**不包含**（后续版本）：
- 用户注册/登录
- 付费功能
- 社交分享
- 移动端优化
- 复杂推荐算法

### 1.3 成功标准
- 用户完成率：70%的用户完成四步选择
- 用户满意度：NPS > 30（10分制）
- 测试反馈：至少3个用户说"这让我更了解自己"

---

## 2. 用户流程

### 2.1 整体流程
```
访问网站 → 看到欢迎页 → 点击"开始灵魂探索" → 第一步选择 → 第二步选择 → 
第三步选择 → 第四步选择 → 生成灵魂画像 → 查看推荐 → 结束
```

### 2.2 页面流程
```
欢迎页 (/) → 选择器页 (/selector) → 结果页 (/result)
```

## 3. 详细功能需求

### 3.1 欢迎页 (Welcome Page)

#### 功能描述
用户进入网站看到的第一个页面，介绍产品并引导开始探索。

#### 需求详情
**UI元素**：
1. **Logo**：SoulCurator + 灵魂策展人（中英文）
2. **主标题**：发现你的文艺灵魂
3. **副标题**：用选择题探索你真正的审美偏好
4. **开始按钮**：大型CTA按钮"开始灵魂探索"
5. **背景**：深色星空渐变，有缓慢移动的粒子效果

**交互**：
- 点击"开始灵魂探索"按钮 → 跳转到选择器页
- 按钮有悬停效果（放大、发光）

**设计要点**：
- 营造神秘、探索的氛围
- 突出"游戏化"感觉
- 响应式设计（PC端优先）

### 3.2 海克斯选择器页 (Selector Page)

#### 功能描述
用户进行四步选择的页面，核心交互界面。

#### 整体布局
```
[顶部进度条]
[当前问题区域]
[选项卡片区域（4张卡片）]
[底部导航按钮]
```

#### 组件1：进度条 (Progress Bar)
**需求**：
- 显示当前步骤（1/4, 2/4, 3/4, 4/4）
- 视觉表现：4个圆点，当前步骤高亮，已完成步骤实心
- 动画：切换步骤时有滑动动画

#### 组件2：问题区域 (Question Area)
**需求**：
- 显示当前问题标题（大字体）
- 显示当前问题描述（小字体）
- 动画：问题切换时有淡入淡出效果

**问题列表**：
1. **第一步**：你的文艺灵魂从何处开始？
   - 描述：选择你最亲近的艺术形式
   
2. **第二步**：[根据第一步动态变化]
   - 电影：什么样的电影最能触动你？
   - 文学：你希望在书中找到什么？
   - 音乐：音乐对你意味着什么？
   - 游戏：你在游戏中寻求什么？
   
3. **第三步**：什么样的美最能打动你？
   - 描述：选择你的审美偏好
   
4. **第四步**：你相信什么？
   - 描述：选择最接近你内心的价值观

#### 组件3：选项卡片 (Option Cards)
**需求**：
- 每次显示4张卡片
- 卡片布局：2×2网格（PC端），1列4行（移动端）
- 卡片内容：
  - 图标（emoji或简单图标）
  - 选项标题（加粗）
  - 选项描述（1-2行简短说明）
- 交互状态：
  - 默认状态：卡片有轻微阴影，背景半透明
  - 悬停状态：卡片放大，阴影加深，有光效
  - 选中状态：卡片边框发光，背景色变化
- 选择逻辑：单选，选中后其他卡片变暗

**选项数据**：
```
第一步选项：
1. 🎬 电影 - 通过画面和故事感受世界
2. 📚 文学 - 通过文字和想象探索内心  
3. 🎵 音乐 - 通过旋律和节奏连接情感
4. 🎮 游戏 - 通过互动和选择体验人生

第二步选项（以电影为例）：
1. 🌌 史诗震撼 - 宏大叙事，人类命运
2. 🧠 思想挑战 - 烧脑剧情，哲学思考
3. 💔 情感深度 - 人性探索，情感细腻
4. 🎭 艺术实验 - 形式创新，视觉独特

第三步选项：
1. 🏛️ 古典优雅 - 秩序、对称、经典之美
2. ⚙️ 现代简约 - 功能、极简、理性之美
3. 🕰️ 复古怀旧 - 时光、记忆、温情之美
4. 🚀 未来科技 - 创新、科幻、未知之美

第四步选项：
1. 🔍 理性探索 - 真理高于情感，逻辑指引方向
2. ❤️ 情感连接 - 爱是终极答案，感受定义真实
3. 🌱 成长进化 - 不断突破自我，生命在于超越
4. 🕊️ 和谐平衡 - 寻找中间道路，万物皆有联系
```

#### 组件4：导航按钮 (Navigation Buttons)
**需求**：
- **上一步按钮**：除第一步外都显示
- **下一步按钮**：始终显示，未选择时禁用
- **完成按钮**：第四步时显示"生成灵魂画像"（替换下一步）

**交互逻辑**：
- 点击选项 → 下一步按钮启用
- 点击下一步 → 保存当前选择，加载下一步问题
- 点击上一步 → 返回上一步，保留之前的选择
- 点击完成 → 跳转到结果页

### 3.3 结果页 (Result Page)

#### 功能描述
展示用户的灵魂画像和个性化推荐。

#### 布局
```
[灵魂画像展示区]
[个性分析报告区]  
[个性化推荐区]
[行动号召区]
```

#### 组件1：灵魂画像展示 (Soul Portrait)
**需求**：
- 显示生成的灵魂画像（雷达图形式）
- 标题："你的文艺灵魂画像"
- 副标题：根据你的选择生成
- 动画：加载时有绘制动画

**画像生成逻辑**：
基于四步选择，计算16个维度的得分：
1. 视觉性 🎨
2. 听觉性 🎵
3. 叙事性 📖
4. 互动性 🎮
5. 情感强度 💔
6. 理性程度 🧠
7. 传统倾向 🏛️
8. 创新倾向 🚀
9. 个人焦点 👤
10. 集体焦点 👥
11. 秩序需求 📐
12. 混乱容忍 🌪️
13. 现实锚定 🌍
14. 幻想倾向 🦄
15. 内向探索 🔍
16. 外向表达 🗣️

**每个选项对应维度调整**（示例）：
- 选择"电影" → 视觉性+2，叙事性+1
- 选择"史诗震撼" → 集体焦点+2，情感强度+1
- 选择"未来科技" → 创新倾向+2，传统倾向-1
- 选择"理性探索" → 理性程度+2，情感强度-1

#### 组件2：个性分析报告 (Personality Analysis)
**需求**：
- 显示灵魂类型名称（如："科幻理性主义者"）
- 显示3-4个关键特征描述
- 显示"原来你是这样的人"的洞察

**报告生成逻辑**：
- 根据最高分的3个维度确定灵魂类型
- 生成对应的特征描述
- 提供"这解释了你为什么喜欢..."的洞察

#### 组件3：个性化推荐 (Personalized Recommendations)
**需求**：
- 显示3个推荐作品
- 每个推荐包含：作品名称、类型、推荐理由
- 推荐理由要连接用户的选择

**推荐逻辑**（v1简化版）：
- 硬编码匹配表：每个选项组合对应3个作品
- 后续版本改为算法推荐

#### 组件4：行动号召 (Call to Action)
**需求**：
- 按钮1："重新探索"（回到欢迎页）
- 按钮2："保存结果"（v1只是提示，实际功能后续版本）
- 按钮3："分享给朋友"（v1只是提示）

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间：< 3秒
- 选择切换响应时间：< 200ms
- 结果生成时间：< 1秒

### 4.2 兼容性需求
- 浏览器：Chrome 90+，Firefox 88+，Safari 14+
- 设备：PC端优先，基础移动端适配
- 屏幕尺寸：支持1920×1080，1366×768，移动端375×667

### 4.3 用户体验需求
- 交互动画流畅（60fps）
- 视觉反馈及时（悬停、点击）
- 错误处理友好（网络错误、数据错误）
- 无障碍基础支持（键盘导航、屏幕阅读器）

## 5. 数据设计

### 5.1 数据结构
```typescript
// 用户选择记录
interface UserSelection {
  step1: OptionId;  // 第一步选择
  step2: OptionId;  // 第二步选择
  step3: OptionId;  // 第三步选择  
  step4: OptionId;  // 第四步选择
  timestamp: number; // 选择时间
}

// 选项定义
interface Option {
  id: string;      // 唯一标识
  step: number;    // 属于第几步
  title: string;   // 选项标题
  emoji: string;   // 图标
  description: string; // 描述
  dimensions: DimensionScores; // 对应维度得分
}

// 维度得分
interface DimensionScores {
  visual: number;      // 视觉性
  auditory: number;    // 听觉性
  narrative: number;   // 叙事性
  interactive: number; // 互动性
  emotional: number;   // 情感强度
  rational: number;    // 理性程度
  traditional: number; // 传统倾向
  innovative: number;  // 创新倾向
  individual: number;  // 个人焦点
  collective: number;  // 集体焦点
  order: number;      // 秩序需求
  chaos: number;      // 混乱容忍
  realistic: number;  // 现实锚定
  fantasy: number;    // 幻想倾向
  introspective: number; // 内向探索
  expressive: number; // 外向表达
}
```

### 5.2 数据存储
**v1版本**：仅前端存储
- 使用localStorage保存用户选择
- 页面刷新后数据丢失（可接受，v1目标不是持久化）

**后续版本**：
- 后端数据库存储
- 用户账户系统
- 历史记录查询

## 6. 技术实现

### 6.1 技术栈
- **前端框架**：Vue 3 + TypeScript
- **UI组件库**：Ant Design Vue 5.x
- **状态管理**：Pinia
- **图表库**：ECharts（灵魂画像雷达图）
- **动画库**：GSAP（复杂交互动画）
- **构建工具**：Vite
- **代码规范**：ESLint + Prettier

### 6.2 项目结构
```
src/
├── assets/          # 静态资源
├── components/      # 组件
│   ├── Selector/    # 选择器组件
│   ├── ProgressBar/ # 进度条
│   ├── OptionCard/  # 选项卡片
│   └── SoulPortrait/# 灵魂画像
├── views/           # 页面
│   ├── Welcome.vue  # 欢迎页
│   ├── Selector.vue # 选择器页
│   └── Result.vue   # 结果页
├── stores/          # 状态管理
│   └── selection.ts # 选择状态
├── data/            # 静态数据
│   ├── options.ts   # 选项定义
│   └── dimensions.ts# 维度定义
├── utils/           # 工具函数
│   ├── calculator.ts# 得分计算
│   └── generator.ts # 报告生成
└── router/          # 路由配置
```
