# 项目技术总结

## 后端

2026.3.30:

用龙虾把项目给项目结构搭起来了，它这玩意是真的秀，用一个docker-compose.yml文件直接把项目需要的依赖直接给安装上了，然后我只需要先执行

```bash
docker-compose up -d
```

然后再执行

```bash
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev
```

总结一下项目的启动命令：

```bash
# 1. 启动数据库（如果还没启动）
docker-compose up -d postgres redis

# 2. 启动Spring Boot后端
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev
```

## 前端

引入的框架及作用：

- Tailwind CSS:是一个“实用工具优先”的 CSS 框架，它提供大量低级别的原子类（如 `p-4`、`text-center`、`bg-blue-500`），让你直接在 HTML 中组合样式，无需写自定义 CSS。它非常灵活，适合快速构建定制化界面,主流 AI 编程助手（如 GitHub Copilot、Claude）在生成 UI 代码时，‌**默认输出 Tailwind 类名**‌，因其原子化、可预测的类名结构非常适合 AI 生成。

Claude-code-router:

`Router` 对象定义了在不同场景下使用哪个模型：

- `default`: 用于常规任务的默认模型。
- `background`: 用于后台任务的模型。这可以是一个较小的本地模型以节省成本。
- `think`: 用于推理密集型任务（如计划模式）的模型。
- `longContext`: 用于处理长上下文（例如，> 60K 令牌）的模型。
- `longContextThreshold` (可选): 触发长上下文模型的令牌数阈值。如果未指定，默认为 60000。
- `webSearch`: 用于处理网络搜索任务，需要模型本身支持。如果使用`openrouter`需要在模型后面加上`:online`后缀。
- `image`(测试版): 用于处理图片类任务（采用CCR内置的agent支持），如果该模型不支持工具调用，需要将`config.forceUseImageAgent`属性设置为`true`。
