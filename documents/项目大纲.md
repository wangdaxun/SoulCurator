# SoulCurator（灵魂策展人）

## 项目MVP

```
功能模块           | 描述                          | 技术复杂度
------------------|-------------------------------|-----------
1. 作品管理        | CRUD操作，支持游戏/电影/书籍  | 低
2. 标签系统        | 手动打标签，支持多对多关系    | 中
3. 基础推荐        | 基于标签相似度的简单推荐      | 中
4. 用户界面        | 简洁的列表和详情页           | 低
```

## 📊 功能优先级矩阵（MoSCoW方法）

### Must Have（必须有）：

1. **作品添加/编辑/删除**
2. **标签系统**（支持：主题、情感、风格）
3. **基于标签的相似度推荐**
4. **作品列表和详情查看**

### Should Have（应该有）：

1. **作品搜索和筛选**
2. **个人偏好设置**
3. **数据导出功能**
4. **响应式设计**

### Could Have（可以有）：

1. **豆瓣/Steam API导入**
2. **情感深度评分**
3. **基础数据可视化**
4. **暗色模式**

### Won't Have（这次不会有）：

1. **社交功能**
2. **移动App**
3. **多语言支持**
4. **付费功能**

### 核心用户旅程：

```
1. 登录系统（我觉得这里可以让用户有选择：新用户的话可以自己选是先登陆系统还是先添加第一部自己喜欢的作品，体验一下这款产品的核心旅程，而且最开始是需要吸引用户的，让用户有继续用这个系统的欲望）
   ↓
2. 添加第一部作品（如《最后的生还者》）
   ↓
3. 打标签：[主题:人性黑暗] [情感:沉重] [风格:叙事驱动]
   ↓
4. 添加更多作品，建立个人库
   ↓
5. 查看推荐："基于你的标签，可能喜欢《底特律变人》"
   ↓
6. 探索关联："《三体》和《星际穿越》都探讨了宇宙孤独"
```



### 可能的游戏化元素：

1. **叙事驱动**：像《底特律变人》那样的分支选择
2. **探索感**：像《塞尔达》那样的地图解锁
3. **收集系统**：像《宝可梦》那样的图鉴收集
4. **成就系统**：解锁里程碑和徽章
5. **视觉风格**：赛博朋克/文艺复古/极简艺术

### 具体交互想法：

- **文艺地图**：用游戏地图的形式展示你的探索轨迹
- **情感温度计**：像RPG属性面板一样展示你的文艺偏好
- **故事模式**：添加作品时像在完成一个叙事任务
- **隐藏成就**：发现特定关联时解锁彩蛋
